# Salesforce Payments

## Overview

Salesforce Payments focuses on checkout efficiency and saved-payment access.

## Official Release Note

- [Salesforce Payments](https://help.salesforce.com/s/articleView?id=release-notes.rn_payments.htm&language=en_US)
- [Hide Shipping Fields](https://help.salesforce.com/s/articleView?id=release-notes.rn_comm_262_hide_shipping.htm&language=en_US)
- [OTP Payment](https://help.salesforce.com/s/articleView?id=release-notes.rn_comm_262_otppayment.htm&language=en_US)

The main theme is reducing checkout friction for services and digital goods.

## What changed

- Unnecessary shipping fields can be hidden dynamically for digital goods.
- One-time password authentication helps buyers retrieve saved payment and account details quickly.
- Buyers can save shipping and payment information and use OTP-based login to get that information back faster.

## Notes for Agents

- The key theme is reducing checkout friction.
- If checkout asks for information that a digital-good purchase does not need, this page explains the new hiding behavior.
