# Agentforce Voice

## Overview

Agentforce Voice is now generally available for mobile apps through the Agentforce Mobile SDK.

## Official Release Note

- [Add Voice to Your Mobile App with Agentforce Voice](https://help.salesforce.com/s/articleView?id=release-notes.rn_mobile_afmsdk_voice_ga.htm&language=en_US)

## Related help

- [Integrate Agentforce into Your Mobile Apps Using React Native](https://help.salesforce.com/s/articleView?id=release-notes.rn_mobile_afmsdk_react_native.htm&language=en_US)

The mobile release intro also pairs this with meeting transcription improvements in the Salesforce mobile app.

## What changed

- Voice-to-voice AI agents can be embedded directly into iOS and Android apps.
- React Native support lets teams ship to iOS and Android from one project.
- The feature is now generally available for integration in mobile apps.
- The Salesforce mobile app supports AI-powered meeting transcriptions alongside the voice capability.
- The change turns voice into a deployable mobile SDK capability rather than a demo-only experience.
- The release note also calls out a React Native integration path for teams that want to embed Agentforce voice into cross-platform mobile apps.

## Notes for Agents

- This is the most direct Agentforce runtime feature in Mobile.
- Use this page when you need the mobile-side deployment and SDK context for voice agents.
- It matters most when an app team needs to ship voice agents inside a native or React Native mobile shell.
- Use the related React Native link when the implementation question is about the mobile SDK wiring rather than the voice capability itself.
