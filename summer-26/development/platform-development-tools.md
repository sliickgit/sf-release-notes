Platform development tools center on Salesforce DX and the broader custom app development experience.

## What changed

- Agentforce DX lets you create, preview, and test agents in a Salesforce DX project.
- The Salesforce DX MCP Server exposes natural-language tools for syncing metadata, running Apex and agent tests, and creating scratch orgs.
- Salesforce CLI remains the command-line path for environment creation, source sync, test execution, and lifecycle control.
- The Salesforce Extension pack still covers development orgs, Apex, Lightning Web Components, Aura, and Visualforce in VS Code.
- Agentforce Vibes IDE provides a browser-based IDE with the same general workflow power as VS Code plus Salesforce CLI.
- Web Console is the in-Salesforce browser IDE for code, SOQL, trace flags, debug logs, and anonymous Apex.

## Notes for Agents

- The main theme is that Salesforce is pushing more of the developer workflow into agent-aware and browser-based tooling.
