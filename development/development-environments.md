# Development Environments

## Overview

Development environments are the Salesforce environments used to build and test features and custom applications.

## What changed

- You can now create a non-preview sandbox during the release-transition window without contacting Salesforce Customer Support.
- Preview sandboxes still upgrade about six weeks before production orgs.
- Non-preview sandboxes now become an explicit choice when you want production-like testing without the next release upgrade.

## Notes for Agents

- This is most useful when you need to control sandbox upgrade timing for validation work.
