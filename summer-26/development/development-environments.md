# Development Environments

## Overview

Development environments are the Salesforce environments used to build and test features and custom applications.

## Official Release Note

- [Development Environments](https://help.salesforce.com/s/articleView?id=release-notes.rn_dev_environments.htm&language=en_US)
- [Sandbox Selection](https://help.salesforce.com/s/articleView?id=release-notes.rn_sandbox_non_preview_selection.htm&language=en_US)
- [Data Mask and Seed](https://help.salesforce.com/s/articleView?id=release-notes.rn_data_mask_and_seed.htm&language=en_US)

## What changed

- You can now create a non-preview sandbox during the release-transition window without contacting Salesforce Customer Support.
- Preview sandboxes still upgrade about six weeks before production orgs.
- Non-preview sandboxes now become an explicit choice when you want production-like testing without the next release upgrade.
- The new sandbox-selection flow gives release teams more control over when test orgs pick up the next release.

## Notes for Agents

- This is most useful when you need to control sandbox upgrade timing for validation work.
- Use the Sandbox Selection article when you need the exact setup behavior for preview versus non-preview sandboxes.
