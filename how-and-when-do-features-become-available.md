# How and When Do Features Become Available?

## Overview

Salesforce uses multiple rollout patterns for release features. Some features are available automatically when the release goes live, while others require admin configuration before users can benefit.

## Key distinctions

- Automatic features require no admin action once available.
- Configurable features must be enabled or set up by an admin.
- Some features are beta, pilot, or developer preview and may never reach general availability.
- Release updates can change behavior even when the visible feature name does not change, so this page helps you distinguish product features from enforcement mechanics.

## Notes for Agents

- Use this section to understand deployment risk and user communication needs.
- The rollout model matters as much as the feature description itself.
- When a note mentions preview, beta, or a future enforcement date, read this page before deciding whether the change is ready for production.
