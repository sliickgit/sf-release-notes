# Lightning Components

## Overview

Lightning Components get the largest share of Development updates in Summer '26.

## What changed

- Local Dev is now called Live Preview.
- Single Component Live Preview is generally available, including the Live Preview VS Code extension.
- State managers help centralize and decouple data logic from presentation.
- Lightning Web Security has updated distortions in the distortion viewer.
- Salesforce Release Manager is available in beta for previewing and testing features.
- Lightning Web Components can target API version 67.0.
- New dynamic list capabilities help load large lists without blocking the browser.
- `lightning-combobox` now supports a pill-shaped `button` variant with icons on options.
- `lightning-file-upload` supports files up to 10 GB.
- `lightning-input` adds an `indeterminate` checkbox state.
- `lightning-dual-listbox`, `lightning-tree-grid`, and several other base components have accessibility and right-to-left updates.
- `<details>` elements can use a `name` attribute so only one grouped details block stays open at a time.
- Hot module reloading is faster.

## Notes for Agents

- This is the highest-value section for front-end and LWC work.
- The Local Dev rename is important for documentation and CLI terminology.
