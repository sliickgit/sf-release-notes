# Release Note Changes

## Overview

Salesforce publishes post-release-note changes separately by product. The most recent edits appear first, and the dates indicate the week those changes were made.

This page is most useful when a release note changed after the PDF was extracted or when a broad note needs a later correction to make the implementation impact clear.

The content here is the correction log for release-note entries, so it helps explain when a later edit changes the meaning of an earlier page.

## What to watch for

- Updated availability details for features already announced.
- Added, removed, or corrected release-note entries after publication.
- Product-specific changes that may affect implementation timing.
- Clarifications that turn a broad release-note item into a concrete implementation detail.

## Notes for Agents

- If a release note looks stale, check this section before assuming the feature has changed behavior.
- This is the best place to reconcile the PDF text with later corrections.
- Treat this page as the delta log for the release notes, not as a replacement for the main feature summaries.
