# Previous Release Notes

## Overview

This section points to earlier release-note archives so you can compare Summer '26 with past releases.

It is mainly useful when you need to tell whether a Summer '26 change is new behavior or a continuation of an older rollout.

## How to use this section

- Look up previous feature announcements.
- Compare feature maturity across releases.
- Trace when a capability first appeared or changed.
- Use the archive when a Summer '26 entry references an older API version, a prior release update, or a feature that appeared earlier in preview or beta.
- The archive helps explain whether the current release is introducing something new or continuing an older rollout.

## Notes for Agents

- If a current note looks incomplete, prior releases often provide context.
- Use the archive as supporting material, not as the primary source for current behavior.
- The archive is especially useful for understanding whether a change is new, renamed, or a follow-up to an older feature.
- This page is a comparison tool, not a source of new release behavior.
- It’s most useful when a feature name changed or when the same capability has appeared in more than one release.
