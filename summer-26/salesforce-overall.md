# Salesforce Overall

## Overview

This section contains platform-wide Salesforce changes that affect the overall experience rather than one product line.

It is the broadest catch-all area in the Summer '26 tree, so it should only hold changes that cut across multiple products or the shared platform itself.

The Summer '26 items here are the shared platform changes that show up across several product areas at once.

## What to use it for

- Core platform behavior and shared UX changes.
- Updates that affect many product areas at once.
- Cross-cutting enhancements that should be tracked centrally.
- Foundational changes such as release-wide security defaults, shared runtime behavior, branding or setup changes, and general platform availability updates.

## Notes for Agents

- Use this section as a catch-all for platform updates that do not belong in a product-specific file.
- If a change looks foundational, it probably belongs here.
- When a feature appears across several product areas, use this section to understand the common platform-level change first.
- This section is most useful when you need the shared platform context before reading the product-specific notes.
