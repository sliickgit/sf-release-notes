# Agentforce Voice SIP Routing

## Overview

Agentforce Voice calls can now be routed using Session Initiation Protocol instead of only a PSTN phone number.

## Official Release Note

- [Agentforce Voice SIP Routing](https://help.salesforce.com/s/articleView?id=release-notes.rn_agentforce_voice_with_sip.htm&language=en_US)

## What changed

- Voice calls can be routed to an Agentforce Service agent over the internet using SIP.
- This provides a more flexible and cost-effective telephony path.
- The change replaces the older PSTN-only path for this use case.
- In practice, the feature gives telephony teams a standards-based route for voice agent deployments instead of requiring only PSTN handling.

## Setup Notes

- Applies to Lightning Experience in Enterprise, Performance, Unlimited, and Developer editions with Foundations or Agentforce 1 editions.
- Set up SIP routing from the Agentforce Voice setup page.
- Partner telephony providers can verify SIP support.
- Available starting February 26, 2026.
- Use this page when the question is how to connect a voice agent to telephony infrastructure.
