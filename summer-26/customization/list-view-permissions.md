# List View Permissions

## Overview

Customization adds more granular list view permissions.

## Official Release Note

- [List View Permissions](https://help.salesforce.com/s/articleView?id=release-notes.rn_listviews_share_private.htm&language=en_US)

## What changed

- The new Manage Shared List Views permission lets users share personal list views with roles, groups, and territories they belong to.
- Users with that permission can edit and delete list views shared with them.
- Shared list view editing can be enabled for users without the new permission.
- The permission change gives admins finer control over who can curate and modify shared list views.

## Notes for Agents

- This is the key admin-facing change in the Customization section.
- If list-view sharing behavior changed, this is the permission model to inspect first.
